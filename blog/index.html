octopress进阶

在github上用octopress建立博客

其实就一个命令

rake setup_github_pages

他会询问你的git库地址，所以你要先建立一个名为usernamegithub.com的库名，注意username是你github的名字。。。 你可能还需要

git config --global user.email "you@example.com"
git config --global user.name "Your Name"

还需要上传你的公钥 前提是你要懂一些git

当然还没完 执行rake deploy会建立_deploy的目录，这个目录会和你的github同步 这样安装完以后更新博客 rake deploy 就可以了

Octopress

静态博客系统octopress

octoppress基于jekyll官网 <octtoppress.org>

1.安装

首先安装git和ruby 推荐用rvm安装 随便找个文件夹

git clone git://github.com/imathis/octopress.git octopress
cd octoppress
bundle install #安装依赖包
rake install #安装默认主题
rake preview #运行，127.0.0.1:4000 查看结果

ps：如果你发现运行rake命令时出现

You have already activated rake 0.9.6, but your Gemfile requires rake 0.9.2.2. Prepending `bundle exec` to your command may solve this.

类似的信息请

bundle exec rake xxx

2.基本用法

创建一个post

$ bundle exec rake new_post["octopress"]
mkdir -p source/_posts
Creating new post: source/_posts/2014-04-22-octopress.markdown
$ cat source/_posts/2014-04-22-octopress.markdown 
---
layout: post  #类型
title: "octopress"  标题
date: 2014-04-22 10:43:49 +0800
comments: true
categories: #分类 多个分类用［］包含
---

创建一个page

halfming@halfming-X45VD:~/git/octopress$ bundle exec rake new_page[neetwork]
mkdir -p source/neetwork
Creating new page: source/neetwork/index.markdown
halfming@halfming-X45VD:~/git/octopress$ bundle exec rake new_page[neetwork/basics]
mkdir -p source/neetwork/basics
Creating new page: source/neetwork/basics/index.markdown
halfming@halfming-X45VD:~/git/octopress$ bundle exec rake new_page[neetwork/basics.html]
mkdir -p source/neetwork
Creating new page: source/neetwork/basics.html
halfming@halfming-X45VD:~/git/octopress$ cat source/neetwork/index.markdown 
---
layout: page
title: "neetwork"
date: 2014-04-22 10:50
comments: true
sharing: true
footer: true
---

其他

rake generate   #生成页面到public文件夹
rake watch      # 查看source和sass文件夹变动
rake preview    # 本地开启web http://localhost:4000

3.配置

_config.yml       # 主配置文件 (Jekyll's settings)
Rakefile          # Configs for deployment
config.rb         # Compass 配置
config.ru         # Rack 配置
